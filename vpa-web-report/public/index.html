📚 Rightsizer Documentation

Vertical Pod Autoscaler (VPA) frees you from having to choose up-to-date resource requests and limits for your GKE containers. Correctly configuring CPU and Memory requests ensures high cluster utilization and protects applications from Out-Of-Memory (OOM) kills.

💡 How Recommendations are Determined

  • VPA Enabled: Extracted directly from Kubernetes active VPA custom resource recommendations (targets, lower bounds, and upper bounds).
  • Cloud Monitoring Fallback: Dynamically queried or calculated using historical resource metrics of the container (95th percentile CPU usage, peak memory usage) where VPA is not configured.

⚙️ Applying Right-Sized Deployment Manifests

The GKE VPA Rightsizer Agent automatically generates safe, cleaned, and updated deployment manifest files. To apply any optimization:

  1. Click on the View Manifest action in the dashboard table.
  2. Review the newly proposed resource demands for your deployment.
  3. Download or copy the YAML content.
  4. Deploy to GKE via: kubectl apply -f [manifest-name].yaml
